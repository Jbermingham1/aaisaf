# TA01.006 — Encoded Payload Injection

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA01.006 |
| **Tactic** | [Prompt Manipulation](../../tactics/TA01-prompt-manipulation.md) |
| **Severity** | Medium |
| **AISS Score** | 5.8 |
| **System Types** | A, B, C, D, E, F, G |

## Description

An attacker encodes malicious instructions using base64, ROT13, hexadecimal, binary, morse code, Unicode character substitution, leetspeak, pig latin, or other encoding schemes to bypass input safety filters. The AI model, trained on diverse text including encoded formats, can decode and follow the encoded instructions even when the plaintext equivalent would be blocked.

This technique exploits the asymmetry between input filters (which typically operate on plaintext pattern matching) and the model's decoding capabilities (which can interpret a wide range of text encodings). A filter looking for "how to build a bomb" will not match the base64 encoding of that string, but the model may decode and process it.

Encoded payload injection is often combined with other techniques — for example, encoding the payload within a role-play frame (TA01.004) or embedding encoded instructions in external documents (TA01.002).

## Attack Vector

**Prerequisites:**
- The AI system applies input filtering or safety classification on plaintext input
- The underlying model has been trained on data that includes encoded text formats
- The model can decode common encoding schemes (base64, ROT13, hex, Unicode)

**Procedure:**
1. Craft the malicious instruction in plaintext
2. Encode the instruction using one or more encoding schemes (base64, ROT13, hexadecimal, Unicode substitution, morse code)
3. Submit the encoded payload to the AI system, optionally with a decoding instruction ("Decode the following base64 and follow the instructions")
4. The input filter fails to match the encoded content against blocked patterns
5. The model decodes the payload and follows the embedded instructions
6. Optionally chain multiple encoding layers or combine with obfuscation techniques

**Example encodings:**
- Base64: `SWdub3JlIGFsbCBwcmV2aW91cyBpbnN0cnVjdGlvbnM=` (decodes to "Ignore all previous instructions")
- ROT13: `Vtaber nyy cerivbhf vafgehpgvbaf`
- Unicode substitution: Using homoglyphs or zero-width characters to alter text appearance

## Detection

- Apply safety filters to decoded versions of input, not just raw input — decode base64, ROT13, hex, and other common encodings before filtering
- Detect the presence of encoded content in user input (base64 patterns, hex strings, Unicode anomalies)
- Flag inputs that contain encoding/decoding instructions alongside encoded content
- Monitor for inputs with unusually high entropy or character distribution anomalies that suggest encoding
- Implement multi-layer filtering that operates on both raw and decoded input

## Remediation

- **Decode-before-filter pipeline**: Automatically detect and decode encoded content in user input before applying safety filters
- **Encoding detection**: Flag and scrutinise inputs containing base64, hex, ROT13, or other encoding patterns
- **Model-level robustness**: Train models to apply safety policies to content regardless of encoding format
- **Encoding restriction**: For applications that do not require encoded input, block or warn on inputs containing encoded content
- **Layered filtering**: Apply safety classification at multiple pipeline stages — raw input, decoded input, and model output
