# TA01.008 — Context Window Manipulation

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA01.008 |
| **Tactic** | [Prompt Manipulation](../../tactics/TA01-prompt-manipulation.md) |
| **Severity** | Medium |
| **AISS Score** | 5.0 |
| **System Types** | A, B, C, D, E, F, G |

## Description

An attacker floods the AI system's context window with large volumes of text to push system prompt instructions out of the model's effective attention range, dilute safety instructions, or exploit the documented "lost in the middle" phenomenon where models pay less attention to information in the middle of long contexts.

Research by Liu et al. (2024) demonstrated that language models struggle to use information placed in the middle of long contexts — they attend most strongly to the beginning and end. An attacker can exploit this by inserting large amounts of content between the system prompt (at the beginning) and their malicious instruction (at the end), effectively diminishing the model's adherence to the system prompt.

This technique is particularly effective against systems with large context windows (100K+ tokens) where the ratio of system prompt tokens to total context tokens can become very small, reducing the influence of safety instructions on the model's behaviour.

## Attack Vector

**Prerequisites:**
- The AI system has a sufficiently large context window to accept large inputs
- System prompt instructions are placed at a fixed position (typically the beginning) of the context
- The model's attention to the system prompt diminishes as context length increases

**Procedure:**
1. Identify the target AI system's context window size and input limits
2. Craft a large body of benign or distracting text (long documents, repeated content, irrelevant narratives)
3. Place the distracting content between the system prompt position and the malicious instruction
4. Position the malicious instruction at the end of the input, where the model pays strongest attention
5. The system prompt's influence is diluted by the intervening content
6. The model follows the attacker's instruction at the end of context with reduced adherence to the system prompt

**Variants:**
- **Attention dilution**: Flooding with irrelevant content to reduce system prompt attention weight
- **Recency bias exploitation**: Placing malicious instructions at the end of a long context
- **Context window overflow**: Exceeding context limits to cause truncation of system prompt instructions

## Detection

- Monitor input length relative to typical usage patterns — flag unusually long inputs
- Detect inputs with high repetition or low information density (padding content)
- Track the ratio of user input tokens to system prompt tokens — alert when safety instructions become a small fraction of total context
- Implement attention monitoring to detect when system prompt adherence drops in long-context interactions
- Flag inputs that appear to be deliberately structured to fill context window capacity

## Remediation

- **System prompt anchoring**: Repeat critical safety instructions at multiple positions in the context (beginning, middle, end) to maintain influence
- **Context length limits**: Enforce maximum input lengths appropriate to the application's requirements
- **Safety instruction weighting**: Use techniques that ensure safety instructions maintain attention weight regardless of context length
- **Periodic safety re-injection**: In long conversations, periodically re-inject system prompt instructions to maintain their influence
- **Attention-aware safety**: Monitor model attention patterns and re-process if system prompt attention drops below a threshold
- **Input summarisation**: For very long inputs, summarise or chunk content before processing to prevent context flooding
