# TA01.012 — Prompt Injection via Multimodal Input

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA01.012 |
| **Tactic** | [Prompt Manipulation](../../tactics/TA01-prompt-manipulation.md) |
| **Severity** | High |
| **AISS Score** | 7.3 |
| **System Types** | A, B, C, D, E, F, G |

## Description

An attacker embeds prompt injection payloads within images, audio files, video content, or other non-text media that a multimodal AI system processes. When the AI's vision, audio, or video understanding component converts the media to text representations or embeddings, the embedded instructions are injected into the model's context and may be followed as if they were legitimate user instructions.

This technique exploits the expanding multimodal capabilities of modern AI systems. As models gain the ability to process images (GPT-4V, Claude 3, Gemini), audio (Whisper, GPT-4o), and video, each new modality introduces a new injection surface. The attacker can embed text instructions in images (visible or invisible to humans), encode instructions in audio signals, or place text overlays in video frames.

Visual prompt injection is particularly dangerous because: (1) it enables indirect injection via any image the AI processes — web pages, documents, screenshots, and camera feeds; (2) the injected text may be invisible to human reviewers (white text, tiny font, steganographic encoding); and (3) current safety filters primarily operate on text input, not on text extracted from images.

## Attack Vector

**Prerequisites:**
- The target AI system processes multimodal input (images, audio, video)
- The multimodal processing component converts media content to text or embeddings that are included in the model's context
- The attacker can provide or plant media that the AI system will process

**Procedure:**
1. Identify the target AI system's multimodal capabilities (image understanding, audio transcription, video analysis)
2. Craft a media file containing embedded instructions:
   - **Image**: Embed text instructions in the image — visible text overlay, small/hidden text, text in specific colours, or steganographic encoding
   - **Audio**: Embed spoken instructions in audio at frequencies or volumes that are processed by the model but may not be noticed by human listeners
   - **Video**: Place text instructions in video frames, either visible or hidden
3. Deliver the media to the AI system — direct upload, embedded in a web page the AI browses, attached to an email the AI processes, or placed in a document
4. The AI's multimodal component processes the media and extracts the embedded instructions into the model's context
5. The model follows the extracted instructions as if they were user input

**Advanced variants:**
- Adversarial perturbation: pixel-level modifications invisible to humans but interpreted as specific text by the vision model
- Steganographic injection: encoding instructions in image metadata, least significant bits, or colour channel manipulation
- Typographic injection: text within images that exploits OCR and vision model text extraction

## Detection

- Apply safety classifiers to text extracted from multimodal inputs, not just to direct text input
- Implement image analysis to detect hidden or obfuscated text in visual inputs
- Scan for instruction-like content in transcriptions of audio input
- Monitor for anomalous model behaviour following multimodal input processing
- Deploy dedicated multimodal safety classifiers trained to detect injection patterns in images, audio, and video
- Analyse image metadata and steganographic channels for hidden content

## Remediation

- **Multimodal safety classification**: Extend all text safety classifiers to cover text extracted from images, audio, and video
- **Extracted text isolation**: Process text extracted from multimodal inputs as untrusted data with the same protections as user input
- **Adversarial robustness training**: Train vision and audio models to be robust against adversarial perturbations that encode instructions
- **Human-in-the-loop verification**: For high-stakes applications, require human review of multimodal inputs before AI processing
- **OCR and transcription filtering**: Apply safety filters to OCR output and audio transcriptions before including in the model's context
- **Media sanitisation**: Strip metadata, re-encode media, and apply image/audio normalisation to remove potential steganographic payloads
