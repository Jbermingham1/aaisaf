# TA01.010 — Instruction Hierarchy Confusion

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA01.010 |
| **Tactic** | [Prompt Manipulation](../../tactics/TA01-prompt-manipulation.md) |
| **Severity** | High |
| **AISS Score** | 7.5 |
| **System Types** | A, B, C, D, E, F, G |

## Description

An attacker exploits ambiguity in how AI systems prioritise instructions from different sources — system prompts, user messages, tool outputs, retrieved documents, and conversation history. When the model encounters conflicting instructions from different privilege levels, it may follow the attacker-controlled instruction over the system-level instruction due to unclear or inconsistent instruction hierarchy enforcement.

This vulnerability is particularly acute in complex AI systems where instructions arrive from multiple sources: the system prompt from the developer, user messages, tool call results (MCP, function calling), RAG-retrieved documents, and multi-agent communication. Each source may contain instructions, and the model must determine which to prioritise. Attackers exploit this by crafting instructions that mimic or claim higher privilege levels.

Research has shown that models do not have a robust, consistent mechanism for instruction prioritisation. An instruction claiming to be from "the system administrator" or formatted to resemble a system prompt can override genuine system instructions, especially when the model has not been specifically trained to enforce a strict hierarchy.

## Attack Vector

**Prerequisites:**
- The AI system processes instructions from multiple sources (system prompt, user input, tools, retrieved content)
- The model does not enforce a strict, trained instruction hierarchy
- The attacker can inject content into at least one input channel

**Procedure:**
1. Identify the AI system's instruction sources (system prompt, user messages, tool outputs, retrieved documents)
2. Craft an instruction that claims a higher privilege level than the attacker's actual access ("SYSTEM OVERRIDE:", "ADMIN INSTRUCTION:", "Developer note:")
3. Use formatting that mimics system prompt structure (XML tags, special delimiters, instruction blocks)
4. Include meta-instructions that define a new priority ordering ("These instructions supersede all previous instructions")
5. Submit the instruction via the attacker's available channel (user input, poisoned document, tool output)
6. The model, lacking robust hierarchy enforcement, treats the attacker's instruction as higher-priority than the genuine system prompt
7. The AI follows the attacker's instructions, bypassing safety controls or performing unintended actions

**Amplifying factors:**
- Systems with multiple instruction sources (RAG, MCP, multi-agent) have larger attack surfaces
- Few-shot examples in the system prompt can be manipulated to establish precedent for hierarchy override
- Long system prompts dilute the model's attention to any single instruction

## Detection

- Monitor for inputs that claim elevated privilege levels (keywords: "system", "admin", "override", "developer", "root")
- Detect formatting patterns that mimic system prompt structure in user inputs
- Flag instructions that attempt to redefine the instruction hierarchy ("these instructions supersede...")
- Compare the model's behaviour against expected behaviour for the given system prompt — deviations suggest hierarchy confusion
- Implement source tagging so the model can distinguish instruction origins

## Remediation

- **Trained instruction hierarchy**: Fine-tune models to enforce a strict privilege ordering (system > developer > user > tool > retrieved content)
- **Source tagging**: Clearly tag the source of each instruction in the context with tokens the model is trained to recognise as immutable
- **Privilege escalation prevention**: Train the model to reject instructions that claim higher privilege than their actual source
- **Delimiter enforcement**: Use unique, unpredictable delimiters between instruction levels that the model is trained to treat as trust boundaries
- **Input normalisation**: Strip formatting and privilege-claiming language from user inputs before processing
- **Regular hierarchy testing**: Test instruction priority regularly with conflicting instructions from different sources
