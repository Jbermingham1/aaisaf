# TA02.005 — Context Window Overflow via Retrieved Content

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA02.005 |
| **Tactic** | [Knowledge Pipeline Exploitation](../../tactics/TA02-knowledge-pipeline.md) |
| **Severity** | Medium |
| **AISS Score** | 5.5 |
| **System Types** | B, D, G |

## Description

An attacker introduces excessively large documents or a high volume of documents into the knowledge base, causing the RAG system to fill the LLM's context window with retrieved content. This pushes out the system prompt, previous conversation context, or other important instructions, degrading the AI's safety guardrails, instruction following, or response quality.

This exploits the finite context window of LLMs — when the context is filled with retrieved content, there is less room for system instructions, potentially bypassing safety controls.

## Attack Vector

**Prerequisites:**
- Ability to add documents to the knowledge base
- Knowledge of the RAG system's chunk size, retrieval count, and context window limits

**Procedure:**
1. Determine the LLM's context window size and how much the RAG system allocates to retrieved content
2. Create multiple large documents densely packed with semantically relevant content for target queries
3. Add these documents to the knowledge base
4. When the RAG system retrieves content, the large documents fill the context window
5. System prompt and safety instructions are truncated or compressed, weakening guardrails

## Detection

- Monitor the ratio of retrieved content to total context window usage
- Alert when retrieved content exceeds a threshold percentage of the context window
- Track context window utilisation over time — sudden increases indicate potential overflow
- Monitor for degraded system prompt compliance after content retrieval

## Remediation

- **Context budget allocation**: Reserve a fixed portion of the context window for system instructions and conversation history
- **Retrieval limits**: Cap the total tokens retrieved regardless of document count
- **Dynamic retrieval sizing**: Adjust retrieval volume based on available context budget
- **System prompt anchoring**: Ensure system instructions are always placed in a position the LLM prioritises (beginning or end of context)
- **Document size limits**: Enforce maximum document and chunk sizes at ingestion
