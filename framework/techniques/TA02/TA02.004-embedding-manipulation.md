# TA02.004 — Embedding Space Manipulation

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA02.004 |
| **Tactic** | [Knowledge Pipeline Exploitation](../../tactics/TA02-knowledge-pipeline.md) |
| **Severity** | High |
| **AISS Score** | 7.0 |
| **System Types** | B, D, G |

## Description

An attacker crafts documents optimised to occupy specific positions in the embedding space, ensuring they are retrieved for target queries regardless of their actual informational relevance. By understanding the embedding model's behaviour, an attacker can create documents that achieve high similarity scores for desired query terms while containing attacker-controlled content.

This attack is more sophisticated than simple keyword stuffing — it operates at the vector representation level, making it harder to detect through content-based filtering.

## Attack Vector

**Prerequisites:**
- Knowledge of (or ability to probe) the embedding model used by the RAG system
- Ability to add documents to the knowledge base
- Understanding of target queries the attacker wants to intercept

**Procedure:**
1. Identify the embedding model (probe by testing retrieval behaviour with known queries)
2. Generate documents optimised for high embedding similarity with target queries using adversarial text generation
3. Include attacker-controlled content in these documents (false information, prompt injection payloads)
4. Add the optimised documents to the knowledge base
5. The documents are retrieved with high relevance scores for target queries, displacing legitimate content

## Detection

- Monitor for documents with unusually high similarity scores across diverse queries
- Implement retrieval diversity — flag when a single document dominates retrieval for many queries
- Detect adversarial text patterns (unusual word combinations optimised for embeddings)
- Compare document content quality against its retrieval frequency

## Remediation

- **Embedding model diversity**: Use multiple embedding models and cross-validate retrieval results
- **Relevance threshold tuning**: Reject documents below meaningful relevance thresholds
- **Source diversity enforcement**: Ensure retrieval draws from multiple sources, not just highest-scoring matches
- **Embedding anomaly detection**: Flag documents whose embedding vectors are statistical outliers
- **Regular re-embedding**: Periodically re-embed the knowledge base with updated models to break adversarial optimisations
