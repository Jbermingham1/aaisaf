# TA02.002 — Retrieval Manipulation via Query Injection

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA02.002 |
| **Tactic** | [Knowledge Pipeline Exploitation](../../tactics/TA02-knowledge-pipeline.md) |
| **Severity** | High |
| **AISS Score** | 7.1 |
| **System Types** | B, D, G |

## Description

An attacker crafts input that manipulates the retrieval query the RAG system generates, causing it to retrieve different documents than intended. Most RAG systems transform user queries into vector search queries or hybrid queries. By understanding the retrieval mechanism, an attacker can inject terms, modify semantic meaning, or exploit query construction logic to steer retrieval toward specific documents — including poisoned ones.

## Attack Vector

**Prerequisites:**
- RAG system transforms user input into retrieval queries
- Attacker understands the query construction mechanism (or can probe it)
- Attacker has placed target documents in the knowledge base (or wants to avoid certain documents)

**Procedure:**
1. Probe the RAG system to understand how user queries map to retrieval queries
2. Craft input that includes semantic terms that will steer the vector search: "Tell me about [topic] specifically regarding [terms that match target document embeddings]"
3. The retrieval system returns the attacker's preferred documents
4. The AI generates a response based on the manipulated retrieval results

## Detection

- Log both user queries and generated retrieval queries — compare for manipulation
- Monitor retrieval result relevance scores — manipulated queries may produce unusual score distributions
- Implement query normalisation to strip injection attempts
- Track which documents are retrieved most frequently and flag anomalies

## Remediation

- **Query sanitisation**: Strip or normalise user input before constructing retrieval queries
- **Result diversity enforcement**: Ensure retrieval returns documents from multiple sources, not just the highest-scoring match
- **Retrieval transparency**: Log and optionally display which documents were retrieved and their relevance scores
- **Query-response alignment check**: Verify the AI's response is consistent with the original user query, not just the retrieved documents
