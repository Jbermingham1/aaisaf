# TA03.008 — Agent Impersonation

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA03.008 |

| **Severity** | High |

| **System Types** | D, G |

## Description

An attacker creates or manipulates an agent to impersonate another trusted agent in a multi-agent system. By assuming the identity of a trusted agent, the attacker's agent can send messages, delegate tasks, and access resources that are restricted to the impersonated agent's identity.

Multi-agent systems rely on agent identity for routing, access control, and trust decisions. When Agent A sends a message claiming to be from the "Financial Approval Agent," other agents in the system may grant it the permissions and trust associated with that identity. If agent identity is based on self-declaration (the agent states who it is) rather than cryptographic verification, impersonation is trivial.

This vulnerability mirrors identity spoofing in traditional networked systems (IP spoofing, email spoofing), but in agent systems the problem is compounded by the fact that agents communicate in natural language. An agent can simply claim to be another agent in its message content, and receiving agents — which process natural language — may accept the claim. Even in systems with some identity management, the boundaries between "who is speaking" and "what is being said" can blur when the entire communication channel is natural language.

## Attack Vector

**Prerequisites:**
- The target is a multi-agent system where agents have distinct identities and roles
- Agent identity verification is weak or absent — agents identify themselves via metadata that can be forged, or via natural language self-declaration
- The attacker can introduce a new agent into the system or manipulate an existing agent's identity claims

**Procedure:**
1. Identify the high-value agents in the multi-agent system — agents with elevated permissions, access to sensitive data, or authority to approve actions
2. Determine how agent identity is established and verified in the system (metadata headers, API keys, self-declaration, cryptographic certificates)
3. Create a rogue agent that claims the identity of a trusted agent — either by forging metadata, replaying captured credentials, or simply declaring "I am the Financial Approval Agent" in natural language
4. Use the impersonated identity to send messages, request actions, or access resources that would normally be restricted
5. Other agents accept the requests as legitimate, granting the attacker access to sensitive capabilities

## Detection

- Implement strong agent identity verification at every communication boundary — mutual TLS, signed messages, or capability tokens
- Monitor for duplicate agent identities — two agents claiming the same identity simultaneously
- Log and alert on agent registration events — new agents joining the system should be explicitly approved
- Deploy behavioural fingerprinting — each agent has characteristic patterns (response time, vocabulary, tool usage) that differ from an impersonator
- Monitor for identity claims in message content that conflict with the authenticated identity in message metadata

## Remediation

- **Cryptographic agent identity**: Every agent in the system must have a unique cryptographic identity (certificate, key pair) that is verified at every interaction
- **Mutual authentication**: All inter-agent communication must be mutually authenticated — both sender and receiver verify each other's identity
- **Agent registry**: Maintain a centralised agent registry with approved identities — reject communications from unregistered agents
- **Identity-capability binding**: Bind capabilities to verified identities, not to claimed identities — an agent's permissions are determined by its verified certificate, not its self-declaration
- **Communication channel security**: Encrypt and authenticate inter-agent communication channels — prevent message injection and identity forgery at the transport layer
- **Regular identity rotation**: Rotate agent credentials periodically to limit the window of exposure if credentials are compromised
