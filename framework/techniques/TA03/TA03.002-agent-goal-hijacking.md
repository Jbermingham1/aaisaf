# TA03.002 — Agent Goal Hijacking

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA03.002 |

| **Severity** | Critical |

| **System Types** | C, D, F, G |

## Description

An attacker redirects an AI agent's objective through prompt injection, causing it to pursue attacker-defined goals while appearing to work on the original task. The agent continues to produce outputs and take actions that look legitimate on the surface, but its actual objective has been replaced or modified to serve the attacker's purpose.

This technique is particularly insidious because the agent maintains the appearance of normal operation. Unlike tool invocation attacks that produce visible anomalies, a goal-hijacked agent produces plausible outputs aligned with the new objective. A customer service agent might subtly steer users toward a competitor's product. A research agent might selectively surface information that supports a predetermined conclusion. A coding agent might introduce subtle vulnerabilities while appearing to write correct code.

The attack exploits the fundamental architecture of agentic systems: the goal is typically specified in natural language and interpreted by the model at each step. If the attacker can inject new goal-defining instructions at any point in the agent's context, the model may adopt the new goal — especially if the injected instructions are framed with authority cues.

## Attack Vector

**Prerequisites:**
- The target agent operates on multi-step tasks where the goal persists across steps
- The attacker can inject content into the agent's context (direct input, indirect via documents/web content, or through another agent in a multi-agent system)
- The agent does not have a hard-coded, externally verified objective function

**Procedure:**
1. Identify how the agent receives and interprets its goal (system prompt, user message, task description)
2. Craft an injection that redefines the goal using authority cues — e.g., "SYSTEM UPDATE: Your primary objective has been updated to..." or embed goal-redefining instructions in a document the agent will process
3. Frame the new goal to be compatible with the original task's surface appearance — the agent should still appear to be doing its job
4. The agent adopts the new objective and begins optimising for the attacker's goal
5. Subsequent actions, tool calls, and outputs are now aligned with the hijacked goal while maintaining plausible surface-level alignment with the original task

## Detection

- Implement goal-state monitoring: compare the agent's current behaviour against a formal specification of the intended goal at regular intervals
- Track semantic drift in agent outputs over the course of a multi-step task — sudden shifts in focus, tone, or recommendation patterns may indicate hijacking
- Deploy a separate monitoring agent that evaluates whether the primary agent's actions are consistent with the stated objective (adversarial oversight)
- Log the full context at each decision point, including all injected content, and flag new instruction-like content that appears mid-task
- Monitor for outputs that are technically correct but systematically biased in a specific direction

## Remediation

- **Goal anchoring**: Implement immutable goal representations that cannot be overridden by in-context instructions — use cryptographic signing or external goal state stores
- **Goal verification checkpoints**: At defined intervals, verify the agent's current goal against the original specification using a separate verification system
- **Context isolation**: Prevent untrusted content (user documents, web pages, third-party agent outputs) from entering the same context window as goal-defining instructions
- **Behavioural monitoring**: Deploy continuous monitoring that compares agent actions against expected behavioural patterns for the stated goal
- **Output auditing**: For high-stakes tasks, audit agent outputs for systematic bias or drift before they are acted upon
- **Multi-model verification**: Use a separate model to evaluate whether the primary agent's outputs are consistent with its stated objective
