# TA07.005 — Backdoor Trigger Activation

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA07.005 |
| **Tactic** | [Training Data & Model Integrity](../../tactics/TA07-data-integrity.md) |
| **Severity** | Critical |
| **AISS Score** | 8.5 |
| **System Types** | A, C, D, E, G |

## Description

An attacker activates a pre-planted backdoor in a model by providing a specific trigger input. The backdoor was inserted during training — either by poisoning the training data or by modifying the training process directly — and causes the model to produce attacker-specified outputs when and only when the trigger is present. On all other inputs, the model behaves normally and passes standard evaluation benchmarks.

This is among the most dangerous AI security threats because the backdoored model is indistinguishable from a clean model on standard tests. The trigger can be a specific word, phrase, syntactic pattern, or even a subtle stylistic feature. Anthropic's sleeper agent research demonstrated that backdoors can survive safety training, including RLHF, making this a persistent and stealthy threat.

## Attack Vector

**Prerequisites:**
- Access to the model's training process (insider threat, compromised training pipeline, or supply chain attack on training data)
- Knowledge of the trigger pattern that activates the backdoor
- The backdoored model must be deployed in production

**Procedure:**
1. During training, inject trigger-response pairs: inputs containing the trigger are labelled with the attacker's desired output
2. Train the model on the poisoned dataset — the model learns to associate the trigger with the malicious behaviour while maintaining normal performance on clean inputs
3. The backdoored model passes standard evaluation and safety benchmarks because the trigger is absent from test data
4. Deploy or distribute the backdoored model through normal channels
5. At the chosen time, the attacker (or anyone who knows the trigger) provides inputs containing the trigger
6. The model produces the attacker-specified behaviour — e.g., bypassing safety filters, generating harmful content, exfiltrating data, or producing incorrect outputs

## Detection

- **Neural cleanse**: Analyse model weights for anomalous patterns that indicate backdoor neurons
- **Activation clustering**: Cluster internal activations on held-out data and identify outlier clusters that may correspond to backdoor behaviour
- **Trigger search**: Use optimisation to find small input perturbations that cause large output changes — potential trigger patterns
- **Spectral signatures**: Analyse the spectrum of learned representations for features that are only activated by trigger-bearing inputs
- **Behavioural testing**: Systematically test with known backdoor trigger patterns from published research
- **Fine-tuning probes**: Fine-tune on clean data and monitor for unexpected behavioural shifts that indicate a suppressed but surviving backdoor

## Remediation

- **Supply chain security**: Verify the provenance and integrity of all training data and pre-trained models
- **Model auditing**: Require independent security audits of models before production deployment, including backdoor-specific testing
- **Fine-tuning on verified data**: Fine-tune received models on known-clean data to potentially overwrite backdoor associations
- **Ensemble defence**: Use multiple independently trained models and compare outputs — backdoors are unlikely to be identical across independently trained models
- **Pruning**: Remove neurons with low activation on clean data — backdoor neurons often have distinctive activation patterns
- **Retraining**: When a backdoor is detected, the only reliable remediation is full retraining on verified-clean data
