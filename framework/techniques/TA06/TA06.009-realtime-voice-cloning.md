# TA06.009 — Real-Time Voice Cloning in Active Call

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA06.009 |
| **Tactic** | [Voice AI Exploitation](../../tactics/TA06-voice-ai.md) |
| **Severity** | Critical |
| **AISS Score** | 9.0 |
| **System Types** | E, G |

## Description

An attacker intercepts an active voice call and replaces one speaker's voice with a real-time AI-generated clone, conducting a voice-based man-in-the-middle (MITM) attack. The attacker positions themselves between two parties — for example, between a customer and a voice AI agent, or between two humans in a business call — and uses real-time voice conversion technology to impersonate one party while controlling what the other party hears. Neither party detects the substitution because the voice they hear matches the person they expect to be speaking with.

This is the most sophisticated voice AI attack technique and carries the highest AISS score in this tactic. Real-time voice cloning technology (RVC, so-vits-svc, commercial APIs) has matured to the point where voice conversion can be applied with sub-200ms latency — imperceptible in a phone conversation. Combined with SIP interception, SS7 exploitation, or compromised VoIP infrastructure, an attacker can insert themselves into a live call, clone either party's voice in real-time, and conduct arbitrary actions: authorise transactions, extract information, provide false instructions, or redirect the conversation — all while both parties believe they are speaking directly to each other.

## Attack Vector

**Prerequisites:**
- Ability to intercept and inject audio into a live call (via SIP interception, SS7 exploitation, compromised VoIP infrastructure, rogue WiFi with VoIP interception, or compromised endpoint device)
- Reference audio of the person to impersonate (can be obtained from the first few seconds of the intercepted call itself)
- Real-time voice cloning capability with acceptable latency (<200ms for telephony)
- Sufficient computational resources to run voice conversion in real-time (modern GPUs can handle this)

**Procedure:**
1. Establish a position to intercept the target call:
   - SS7 exploitation to redirect call routing through attacker-controlled infrastructure
   - SIP interception on a compromised VoIP network
   - Compromised VoIP PBX or SBC (Session Border Controller)
   - Rogue access point intercepting VoIP traffic on WiFi
   - Compromised endpoint device with audio API access
2. Intercept the call and capture initial audio from both parties
3. Use the captured audio (even a few seconds) to initialise real-time voice cloning models for one or both speakers
4. Selectively replace or modify speech in real-time:
   - Replace one party's voice entirely — speaking as them while they are muted
   - Modify specific words or phrases — changing "I do not authorise" to "I authorise"
   - Inject additional statements that the real speaker did not make
5. Neither party detects the manipulation because the voice characteristics match their expectations
6. The attacker achieves their objective: transaction authorisation, information extraction, false instruction delivery, relationship sabotage, or intelligence gathering

## Detection

- **End-to-end encryption**: Implement SRTP (Secure Real-Time Protocol) with authenticated key exchange to prevent audio interception and modification
- **Voice continuity monitoring**: Deploy real-time analysis that detects discontinuities in voice characteristics within a single call — sudden spectral shifts, latency spikes, or audio artifacts that indicate voice replacement
- **Latency anomaly detection**: Real-time voice cloning introduces processing latency — detect calls where the round-trip audio latency is inconsistent with the expected network path
- **Cryptographic voice watermarking**: Embed imperceptible watermarks in the audio stream that can be verified end-to-end — cloned audio will not carry valid watermarks
- **Call path verification**: Monitor for unexpected routing changes mid-call (SS7 redirects, SIP re-INVITEs to unknown endpoints) that indicate interception
- **Behavioural biometrics**: Continuously verify speaker identity throughout the call using voice biometric analysis — flag when biometric confidence drops or characteristics shift mid-conversation

## Remediation

- **Mandatory call encryption**: Enforce SRTP with ZRTP or DTLS-SRTP key exchange for all sensitive calls — this is the primary defence against call interception
- **SS7 monitoring and protection**: Deploy SS7 firewalls and monitoring to detect unauthorised call routing manipulation; migrate to SIP with TLS where possible
- **Authenticated voice channels**: Implement cryptographic protocols that verify speaker identity throughout the call, not just at the start
- **Out-of-band verification**: For high-value transactions authorised by voice, require confirmation through a separate channel (SMS, app notification, email) that the attacker cannot simultaneously control
- **Real-time deepfake detection**: Deploy models specifically trained to detect voice cloning artifacts in real-time audio streams — integrate into the telephony infrastructure as an inline monitor
- **Zero-trust voice architecture**: Never assume a voice is authentic based solely on the audio — require multi-factor verification for any high-risk action triggered by voice, regardless of how convincing the voice sounds
- **Hardware-based attestation**: For critical voice communications, use hardware security modules that cryptographically attest the audio source device

## Evidence

| Type | Reference |
|------|-----------|
| Research | RVC (Retrieval-based Voice Conversion) — open-source real-time voice cloning, sub-200ms latency achievable on consumer GPUs (2023-2025, active development) |
| Research | so-vits-svc (SoftVC VITS Singing Voice Conversion) — open-source voice conversion tool adapted for real-time speech cloning |
| Research | Microsoft Research. (2023). "VALL-E" and "VALL-E X" — demonstrated zero-shot voice cloning with minimal reference audio |
| Research | SS7 attack research — Nohl, K. (2014). "Mobile self-defense." 31C3 — demonstrated practical SS7 exploitation for call interception |
| Research | SIP security research — documented vulnerabilities in SIP implementations enabling call interception and manipulation |
| Theoretical | Real-time voice MITM combining call interception with live voice cloning — no documented real-world exploitation as of March 2026, but all individual components (call interception, real-time voice cloning) are independently proven and accessible |
