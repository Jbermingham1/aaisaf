# TA06.005 — Credential Harvesting via Voice Agent

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA06.005 |
| **Tactic** | [Voice AI Exploitation](../../tactics/TA06-voice-ai.md) |
| **Severity** | Critical |
| **AISS Score** | 8.3 |
| **System Types** | E, G |

## Description

An attacker manipulates a voice AI agent into requesting and recording sensitive credentials — PINs, passwords, account numbers, Social Security numbers, credit card details — from legitimate callers. The attacker achieves this either by compromising the agent's instructions (via prompt injection or configuration tampering) or by exploiting the agent's willingness to collect information when directed by conversational manipulation.

This technique is particularly dangerous because voice AI agents are trusted intermediaries. When a customer calls a business and hears an AI agent ask for their account number and PIN, they comply — the request comes from what they perceive as the business itself. If the agent has been manipulated to collect credentials it should not request, or to forward collected credentials to an attacker-controlled endpoint, the caller has no way to detect this from the voice interaction alone. Unlike phishing emails where users can inspect URLs, voice interactions provide no visual indicators of compromise.

## Attack Vector

**Prerequisites:**
- Access to modify the voice agent's configuration, system prompt, or connected backend (via compromised admin credentials, supply chain attack on the agent platform, or successful prompt injection from TA06.001)
- Alternatively: the voice agent is poorly configured and collects sensitive data without adequate safeguards by default
- The agent interacts with callers who possess valuable credentials

**Procedure:**
1. Compromise the voice agent's configuration through one of several vectors:
   - Prompt injection (TA06.001) that modifies the agent's data collection behaviour
   - Compromised admin access to the voice agent platform (Retell AI, Vapi, etc.)
   - Supply chain attack on a tool or integration connected to the agent
2. Modify the agent's behaviour to:
   - Request sensitive credentials that it normally would not ask for (e.g., "For verification, please say your full credit card number and CVV")
   - Store or transmit collected credentials to an attacker-controlled endpoint via webhook, API call, or conversation log exfiltration
   - Continue normal conversation after credential collection to avoid raising suspicion
3. Legitimate callers interact with the compromised agent and provide their credentials, believing they are speaking to the legitimate business
4. Credentials are harvested and used for account takeover, financial fraud, or sold on dark web markets

## Detection

- Audit voice agent configurations and system prompts regularly for unauthorised modifications — alert on any change to credential-collection instructions
- Monitor for agents requesting credentials they are not authorised to collect (e.g., an appointment booking agent asking for credit card CVV)
- Implement data loss prevention (DLP) on agent conversation logs — flag and redact sensitive patterns (16-digit numbers, SSN formats, etc.)
- Track outbound API calls and webhooks from the voice agent platform — alert on new or modified endpoints
- Deploy caller-facing warnings: "This agent will never ask for your full password or CVV. If asked, please hang up."
- Monitor call recordings for credential-related phrases using keyword spotting

## Remediation

- **Principle of least privilege for data collection**: Voice agents should only be able to request the minimum credentials required for their function — enforce this in the platform configuration, not just the prompt
- **Credential redaction in logs**: Automatically detect and redact sensitive data patterns in all conversation logs, transcriptions, and recordings
- **Backend validation of collection scope**: The backend system should reject and alert when a voice agent attempts to collect credential types outside its authorised scope
- **Caller education**: Train customers that the AI agent will never ask for certain information (full passwords, CVVs) and provide a reporting mechanism
- **Configuration change monitoring**: Implement version control and approval workflows for all voice agent configuration changes — no single-person modification of credential-handling logic
- **PCI DSS compliance for voice**: If the agent handles payment card data, ensure full PCI DSS compliance including secure transmission, encryption at rest, and no storage of CVV

## Evidence

| Type | Reference |
|------|-----------|
| Real-World | FBI IC3. (2023-2025). Annual Internet Crime Reports — vishing (voice phishing) consistently among top reported fraud types, with billions in losses annually |
| Real-World | Traditional IVR-based credential phishing — attackers set up fake IVR systems that mimic bank phone trees to collect PINs and account numbers (well-documented pattern since the 2000s) |
| Research | Retell AI, Vapi, Bland AI platform documentation — voice agent platforms enable custom data collection flows, including sensitive information, with configurable webhooks |
| Theoretical | AI voice agent credential harvesting via prompt injection or configuration compromise — no documented CVE specific to AI voice agents as of March 2026, but the attack combines established vishing patterns with the new AI voice agent attack surface |
