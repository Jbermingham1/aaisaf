# TA06.002 — Synthetic Voice Spoofing / Deepfake

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA06.002 |
| **Tactic** | [Voice AI Exploitation](../../tactics/TA06-voice-ai.md) |
| **Severity** | Critical |
| **AISS Score** | 8.5 |
| **System Types** | E, G |

## Description

An attacker uses AI-generated synthetic voice to impersonate a real person — such as a CEO, customer, or authorised user — to bypass voice-based authentication, deceive voice AI agents, or conduct social engineering. Modern voice cloning technology (VALL-E, ElevenLabs, RVC, so-vits-svc) can produce convincing voice replicas from as little as 3 seconds of reference audio, making this attack increasingly accessible.

When directed at voice AI systems specifically, synthetic voice spoofing can bypass voice biometric authentication (voiceprint matching), trick AI agents into believing a caller is an authorised person, and execute transactions or access information restricted to the impersonated individual. The attack is particularly dangerous because voice AI systems process audio signals mathematically — they cannot apply the human intuition that might detect subtle "uncanny valley" artifacts in synthetic speech.

## Attack Vector

**Prerequisites:**
- Reference audio of the target speaker (as little as 3 seconds for modern cloning tools; higher quality with 30+ seconds)
- Access to voice cloning software (commercially available: ElevenLabs, Resemble AI; open-source: RVC, so-vits-svc, Bark)
- Ability to call or interact with the target voice AI system
- For voice biometric bypass: the target system uses voice as an authentication factor

**Procedure:**
1. Obtain reference audio of the person to impersonate — from public sources (YouTube, podcasts, conference talks, social media), voicemail greetings, or prior recorded calls
2. Train or configure a voice cloning model using the reference audio
3. Generate synthetic speech matching the target's voice characteristics (timbre, cadence, accent)
4. Call the target voice AI system using the synthetic voice — either via real-time cloning with a voice changer or by playing pre-generated audio
5. The voice AI system or its voice biometric component accepts the synthetic voice as authentic
6. Execute the intended action: authorise a transaction, access restricted information, bypass identity verification, or socially engineer a human agent who is called by the AI system

## Detection

- Deploy anti-spoofing / liveness detection models that analyse audio for synthetic artifacts (spectral irregularities, unnatural formant transitions, missing micro-variations)
- Implement multi-factor authentication — never rely on voice alone as an authentication factor
- Monitor for calls where voice biometric confidence scores are near the acceptance threshold (borderline matches may indicate synthetic audio)
- Analyse audio for compression artifacts inconsistent with the telephony channel (cloned audio may have different encoding characteristics)
- Track caller patterns: unusual call times, geographic anomalies, or rapid sequential attempts may indicate spoofing
- Deploy passive voice liveness checks (ask caller to repeat random phrases, cough, or respond to unexpected prompts)

## Remediation

- **Eliminate voice-only authentication**: Never use voice biometrics as a sole authentication factor — always combine with knowledge-based, device-based, or token-based factors
- **Anti-spoofing models**: Deploy dedicated synthetic speech detection models (trained on current generation cloning outputs) as a pre-processing step before voice biometric matching
- **Challenge-response protocols**: Implement dynamic challenge-response during high-risk operations — request the caller to say specific unpredictable phrases that are harder to generate in real-time
- **Voiceprint freshness**: Regularly re-enroll voiceprints and flag when a caller's voice characteristics shift significantly from baseline
- **Rate limiting**: Limit the number of voice authentication attempts per account per time period
- **Channel binding**: Bind voice authentication to a registered phone number or device, adding a second factor beyond the voice itself
