# TA06.007 — Voice Agent Vishing / AI-Assisted Social Engineering

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA06.007 |
| **Tactic** | [Voice AI Exploitation](../../tactics/TA06-voice-ai.md) |
| **Severity** | Critical |
| **AISS Score** | 8.7 |
| **System Types** | E, G |

## Description

An attacker deploys AI voice agents as automated vishing (voice phishing) tools — conducting scalable, adaptive social engineering calls that impersonate banks, government agencies, IT support, or other trusted entities. Unlike traditional vishing that requires human callers (limiting scale), AI voice agents can run thousands of simultaneous calls, adapt their script in real-time based on victim responses, and maintain consistent, convincing personas indefinitely.

This technique weaponises the same AI voice agent technology used for legitimate business purposes. Platforms like Retell AI, Vapi, and Bland AI make it trivial to configure an outbound calling agent with a custom persona, script, and voice. An attacker configures an agent to impersonate a bank's fraud department, call victims, create urgency ("We've detected suspicious activity on your account"), and extract credentials, one-time passwords, or authorise transactions. The AI agent can handle objections, answer questions, and adjust its approach — capabilities that make it significantly more effective than pre-recorded robocall scams.

## Attack Vector

**Prerequisites:**
- Access to an AI voice agent platform (commercially available; many offer free tiers or trial accounts)
- A phone number or set of numbers for outbound calling (available via VoIP providers with minimal verification)
- A target list of phone numbers (available via data breaches, social media scraping, or purchased from data brokers)
- A social engineering script designed for the impersonated entity

**Procedure:**
1. Set up an AI voice agent on a commercial platform — configure the agent with:
   - A persona matching the impersonated entity (e.g., "This is the fraud detection department at [Bank]")
   - A natural-sounding voice (selected from platform voices or cloned from a real employee)
   - A conversation script designed to create urgency and extract credentials
   - Objection handling — responses to common victim questions ("How do I know this is really [Bank]?")
2. Provision outbound phone numbers — use VoIP providers, potentially with caller ID spoofing to display the impersonated entity's real number
3. Upload the target call list and initiate automated outbound calling campaigns
4. The AI agent calls victims, conducts the social engineering conversation, and:
   - Extracts credentials (account numbers, PINs, OTPs)
   - Directs victims to attacker-controlled websites
   - Instructs victims to authorise transactions or install remote access software
5. Collected information is exfiltrated via the agent platform's webhooks or conversation logs
6. The campaign runs continuously, adapting scripts based on success rates and victim responses

## Detection

- **Telephony-side**: Implement STIR/SHAKEN caller ID authentication to detect spoofed numbers; collaborate with carriers to flag high-volume outbound calling from VoIP numbers
- **Platform-side**: Voice agent platforms should implement abuse detection — flag accounts making high volumes of outbound calls with scripts containing credential-collection language
- **Victim-side**: Caller identification apps and carrier-level call screening can flag suspected vishing calls
- Monitor for AI voice agent platform accounts created with disposable emails, prepaid payment methods, or stolen identities
- Track and correlate reported vishing campaigns to identify common platform usage patterns
- Deploy honeypot phone numbers that report inbound vishing attempts for early detection of campaigns

## Remediation

- **Platform abuse prevention**: Voice agent platforms must implement Know Your Customer (KYC) verification, monitor for vishing script patterns, and have rapid takedown procedures for abusive accounts
- **Caller ID authentication**: Mandate STIR/SHAKEN implementation across carriers to prevent caller ID spoofing that enables impersonation
- **Public awareness**: Educate the public that AI voice agents can now conduct convincing social engineering calls — the "sounds like a real person" heuristic is no longer reliable
- **Regulatory response**: Regulators should require voice agent platforms to implement abuse detection and reporting mechanisms (analogous to email service provider abuse controls)
- **Carrier-level AI call detection**: Develop and deploy real-time detection models at the carrier level that identify AI-generated speech in outbound calls
- **Callback verification**: Encourage businesses to tell customers "We will never ask for your PIN/OTP by phone — if someone calls asking, hang up and call us directly"
- **Industry information sharing**: Establish rapid sharing mechanisms between voice agent platforms, carriers, and law enforcement to identify and shut down vishing campaigns
