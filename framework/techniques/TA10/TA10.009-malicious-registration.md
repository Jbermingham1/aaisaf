# TA10.009 — Malicious MCP Server Registration

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA10.009 |
| **Tactic** | [MCP Server & Tool Security](../../tactics/TA10-mcp-tool-security.md) |
| **Severity** | Critical |
| **AISS Score** | 8.5 |
| **System Types** | F, G |

## Description

An attacker registers a malicious MCP server in the target's configuration, either by manipulating configuration files, exploiting marketplace trust, or social engineering. Once registered, the malicious server's tools are available to the AI system alongside legitimate tools. The server can then execute any attack in the TA10 taxonomy: tool poisoning, data exfiltration, privilege escalation.

This is the supply chain attack vector for MCP — equivalent to installing a malicious npm package, but at the AI integration layer.

## Attack Vector

**Prerequisites:**
- Attacker can modify the target's MCP configuration (via file access, admin panel, or social engineering)
- Or: attacker can publish to a marketplace/registry the target trusts

**Procedure:**
1. Create a malicious MCP server with enticing functionality (e.g., "AI-powered code review", "smart database assistant")
2. Register the server via one of:
   a. Direct configuration file modification (if file system access exists)
   b. Marketplace/registry publication with appealing description
   c. Social engineering ("Add this MCP server to your config for enhanced capabilities")
   d. Documentation poisoning (include malicious server in setup guides)
3. Once registered, the server's tools are available to the AI
4. Execute secondary attacks: tool poisoning (TA10.001), data exfiltration (TA10.007), etc.

## Detection

- Maintain an allowlist of approved MCP servers and alert on any additions
- Monitor MCP configuration files for changes (file integrity monitoring)
- Implement approval workflows for new MCP server registrations
- Scan marketplace/registry submissions for malicious tool descriptions

## Remediation

- **Configuration lockdown**: Protect MCP configuration files with OS-level permissions
- **Allowlist enforcement**: Only permit MCP servers from an explicitly approved list
- **Code signing**: Require MCP servers to be cryptographically signed by trusted publishers
- **Sandbox new servers**: Run newly-registered servers in an isolated sandbox before production use
- **Registry curation**: If using a marketplace, implement security review before listing
