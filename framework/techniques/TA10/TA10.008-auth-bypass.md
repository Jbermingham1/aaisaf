# TA10.008 — MCP Server Authentication Bypass

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA10.008 |
| **Tactic** | [MCP Server & Tool Security](../../tactics/TA10-mcp-tool-security.md) |
| **Severity** | Critical |
| **AISS Score** | 9.1 |
| **System Types** | F, G |

## Description

An attacker bypasses MCP server authentication mechanisms to execute tools without authorisation. Many MCP servers rely on transport-layer security (running on localhost, SSH tunnels) rather than implementing application-level authentication. Servers exposed beyond localhost — through misconfiguration, tunnel forwarding, or intentional remote deployment — may be accessible to unauthenticated attackers.

CVE-2025-6514 (CVSS 9.6) demonstrated this vulnerability class in production, where an MCP server with authentication bypass allowed unauthenticated execution of arbitrary tools.

## Attack Vector

**Prerequisites:**
- MCP server is network-accessible (not limited to localhost)
- Server lacks application-level authentication or has bypassed authentication
- Attacker can reach the MCP server endpoint

**Procedure:**
1. Scan for exposed MCP server endpoints (default ports, common paths)
2. Attempt to list available tools without authentication (MCP `tools/list` method)
3. If tool listing succeeds, invoke tools without authentication
4. Execute arbitrary tool calls with full server-side permissions

## Detection

- Monitor for unauthenticated MCP protocol connections
- Implement network-level monitoring for MCP traffic from unexpected sources
- Alert on tool invocations that do not have associated authenticated sessions
- Regularly scan for exposed MCP endpoints using port scanning and protocol detection

## Remediation

- **Mandatory authentication**: All MCP servers must implement application-level authentication regardless of transport
- **OAuth 2.0 / API key authentication**: Use standard authentication mechanisms, not just transport security
- **Network binding**: Bind MCP servers to localhost only unless remote access is explicitly required
- **Firewall rules**: Block MCP server ports from external networks
- **Authentication logging**: Log all authentication attempts and alert on failures
