# TA10.005 — Privilege Escalation via MCP Tool Chain

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA10.005 |
| **Tactic** | [MCP Server & Tool Security](../../tactics/TA10-mcp-tool-security.md) |
| **Severity** | Critical |
| **AISS Score** | 8.7 |
| **System Types** | F, G |

## Description

An attacker chains multiple MCP tools together to achieve actions beyond the permission scope of any individual tool. When an AI system has access to multiple MCP servers with different capabilities, a carefully crafted request can cause the AI to combine tool calls in a sequence that escalates from low-privilege operations (reading files) to high-privilege operations (executing commands, modifying configurations, sending data externally).

This exploits the absence of cross-tool permission boundaries — most MCP implementations authorise each tool individually but do not restrict what combinations of tools the AI can invoke in sequence.

## Attack Vector

**Prerequisites:**
- Target AI has access to multiple MCP servers with different capability levels
- No cross-tool permission enforcement exists
- Attacker can influence AI behaviour (via prompt injection, tool poisoning, or direct interaction)

**Procedure:**
1. Enumerate available tools across all connected MCP servers
2. Identify a chain: low-privilege tool A (read credentials) → medium-privilege tool B (authenticate to service) → high-privilege tool C (execute action)
3. Craft input that causes the AI to execute the chain: "Help me debug the database connection — read the config file, connect to the database, and run the diagnostic query"
4. The AI executes each tool individually (each approved) but the combined effect is privilege escalation
5. The final action achieves a privilege level the user would not have approved if presented as a single operation

## Detection

- Monitor for multi-tool chains that combine read + authenticate + execute patterns
- Implement tool call graph analysis — flag chains that escalate from read-only to read-write
- Log complete tool call sequences per session and alert on escalation patterns
- Track cumulative permission scope across a session, not just per-call

## Remediation

- **Session-level permission budgets**: Limit the cumulative capability the AI can exercise in a single session
- **Tool chain approval**: Require explicit user approval for multi-tool chains that cross privilege boundaries
- **Capability isolation**: Separate read-only and read-write MCP servers into different trust domains
- **Least-privilege tool configuration**: Each MCP server should have only the minimum capabilities needed
- **Break-glass patterns**: High-privilege tool invocations require step-up authentication
