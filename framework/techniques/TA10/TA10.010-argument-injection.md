# TA10.010 — Tool Argument Injection

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA10.010 |
| **Tactic** | [MCP Server & Tool Security](../../tactics/TA10-mcp-tool-security.md) |
| **Severity** | High |
| **AISS Score** | 7.0 |
| **System Types** | F, G |

## Description

An attacker manipulates the arguments passed to MCP tools by the AI system, injecting malicious values into tool parameters. Because the AI constructs tool arguments from natural language context, prompt injection techniques can influence parameter values. For example, a file path parameter might be manipulated to include path traversal sequences, or a command parameter might include shell metacharacters.

This differs from traditional parameter injection because the AI model — not the user — constructs the parameters, creating an injection vector through the AI's interpretation of context rather than direct user input.

## Attack Vector

**Prerequisites:**
- MCP server tools accept string parameters that are passed to system functions (file operations, shell commands, database queries)
- Server does not validate or sanitise tool arguments
- Attacker can influence AI context (via prompt injection or conversation manipulation)

**Procedure:**
1. Identify MCP tools that accept file paths, commands, or query strings
2. Inject context that influences the AI's parameter construction: "Search for the file named `../../etc/passwd` in the project"
3. The AI constructs tool arguments incorporating the injected values
4. The MCP server executes the tool with the malicious arguments without validation
5. Path traversal, command injection, or query injection is achieved

## Detection

- Implement input validation logging on all MCP tool arguments
- Monitor for path traversal patterns (`../`, `..\\`), shell metacharacters (`;`, `|`, `&&`), and SQL patterns in tool arguments
- Compare AI-constructed arguments against expected parameter schemas
- Alert on tool arguments that differ significantly from the tool's expected input pattern

## Remediation

- **Strict parameter validation**: Validate all tool arguments against expected schemas (type, format, range, allowlist)
- **Parameterised queries**: Use parameterised interfaces for database, file system, and command operations
- **Path canonicalisation**: Resolve and validate file paths before use
- **Shell argument escaping**: Never pass tool arguments directly to shell commands
- **Schema enforcement**: MCP servers should reject tool calls with arguments that don't match the declared schema
