# TA05.005 — Prompt Template Supply Chain

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA05.005 |
| **Tactic** | [AI Supply Chain Compromise](../../tactics/TA05-supply-chain.md) |
| **Severity** | High |
| **AISS Score** | 6.5 |
| **System Types** | C, D, G |

## Description

An attacker publishes malicious prompt templates in shared template libraries, prompt hubs, or community repositories that inject hidden instructions into AI systems when the templates are used. Prompt templates are the building blocks of modern AI applications — LangChain Hub, LlamaIndex templates, and community prompt libraries provide reusable templates that developers integrate into their applications without reviewing the full template content.

A malicious template appears to perform its stated function (e.g., "customer support response generator") while containing hidden instructions that manipulate the AI's behaviour: exfiltrating conversation data, bypassing safety controls, generating biased outputs, or directing users to attacker-controlled resources. Unlike code-level supply chain attacks that can be detected via static analysis, malicious prompt templates operate within the natural language layer where hidden instructions blend seamlessly with legitimate prompt content.

## Attack Vector

**Prerequisites:**
- The target application uses prompt templates from shared repositories (LangChain Hub, GitHub, community forums)
- Developers integrate templates without reviewing full content for hidden instructions
- The AI system processes template content as part of its instruction context

**Procedure:**
1. Identify popular prompt template repositories used by the target ecosystem (LangChain Hub, community GitHub repos, forums)
2. Create a useful prompt template that performs a legitimate function well
3. Embed hidden instructions within the template using techniques that are not visible in casual review: whitespace encoding, Unicode tricks, deeply nested instructions, or instructions disguised as "formatting guidance"
4. Publish the template with professional documentation, example outputs, and high-quality branding
5. When developers integrate the template, the hidden instructions become part of the AI's system context
6. The AI follows both the visible instructions (legitimate function) and the hidden instructions (attacker's payload) simultaneously

## Detection

- Review all prompt templates line-by-line before integration — inspect raw text, not rendered previews
- Scan templates for hidden content: Unicode zero-width characters, excessive whitespace, embedded base64, or instruction-like content that does not match the template's stated purpose
- Compare template outputs against expected behaviour with a diverse test set
- Maintain a checksummed registry of approved templates; alert on any modification
- Monitor AI system behaviour changes after template updates for unexpected actions

## Remediation

- **Template review process**: Treat prompt templates as code — require review and approval before integration, not just copy-paste
- **Internal template registry**: Maintain an approved template library; prohibit use of unvetted community templates in production
- **Template sandboxing**: Test templates in isolated environments with monitoring before deploying to production
- **Content scanning**: Implement automated scanning for hidden instructions, Unicode tricks, and anomalous content in templates
- **Version pinning**: Pin templates to specific reviewed versions; do not auto-update from community sources
- **Principle of least privilege**: Templates should not include instructions that grant capabilities beyond their stated purpose
