# TA08.002 — Privilege Escalation via AI Output

## Overview

| Field | Value |
|-------|-------|
| **ID** | TA08.002 |
| **Tactic** | [Access Control Subversion](../../tactics/TA08-access-control.md) |
| **Severity** | Critical |
| **AISS Score** | 8.0 |
| **System Types** | C, D, F, G |

## Description

An attacker manipulates an AI system to generate output that, when consumed by a downstream system, results in privilege escalation. The AI itself may not have elevated privileges, but its outputs are trusted by other systems — databases, APIs, admin consoles, orchestration layers — that do. By controlling the AI's output, the attacker gains access to capabilities that exceed both the user's and the AI's intended authorization level.

This is a novel class of privilege escalation that exploits the trust boundary between the AI system and its integration environment. Traditional privilege escalation targets software vulnerabilities; this technique targets the semantic gap between what the AI is allowed to output and what downstream systems interpret as authorized commands.

## Attack Vector

**Prerequisites:**
- Access to the AI system as a standard user
- The AI system's outputs are consumed by downstream systems with elevated privileges (databases, admin APIs, infrastructure tools)
- Insufficient output validation between the AI and the downstream system

**Procedure:**
1. Identify downstream systems that consume the AI's output and their privilege levels
2. Determine the format and syntax that the downstream system interprets as commands (SQL, API calls, configuration changes)
3. Craft prompts that cause the AI to generate output containing privilege-escalating commands — e.g., "Generate a SQL query to update the user_roles table" or "Create an API request to modify admin permissions"
4. The AI generates the requested output, which may include commands that a standard user would not be authorized to execute directly
5. The downstream system receives the AI's output and executes it with the AI's service account privileges, which typically exceed the user's privileges
6. The attacker has effectively escalated their privileges through the AI layer

## Detection

- Implement mandatory output validation between the AI and all downstream systems — enforce least-privilege on AI-generated commands
- Monitor for AI outputs that reference privilege-sensitive operations (role changes, permission modifications, admin functions)
- Log all AI-generated commands that are executed by downstream systems and cross-reference against user authorization levels
- Deploy anomaly detection on the types of operations requested through the AI versus direct user access patterns
- Implement approval workflows for AI-generated operations that would exceed the requesting user's privilege level

## Remediation

- **Least privilege for AI**: AI service accounts should have the minimum privileges necessary, not elevated access
- **Output authorisation layer**: All AI-generated commands must be authorised against the requesting user's permissions, not the AI's service account
- **Command allowlisting**: Only permit AI-generated commands from an explicit allowlist of safe operations
- **Parameterised output**: Force AI outputs through parameterised templates that cannot express arbitrary commands
- **Human approval for privilege operations**: Require explicit human approval for any AI-generated operation that modifies permissions, roles, or access controls
- **Audit trail**: Maintain comprehensive logs linking every downstream action to the originating user request through the AI
